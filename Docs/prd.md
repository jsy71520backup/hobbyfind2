# PRD (Product Requirements Document)

## HobbyFind

---

## 1. 제품 상세 설명 (Detailed Product Description)

### 서비스 개요

| 항목     | 내용                                         |
| ------ | ------------------------------------------ |
| 서비스명   | HobbyFind                                  |
| 서비스 타입 | 웹서비스                                       |
| 대상 사용자 | 다양한 취미를 탐색하고 정리하고 싶은 일반 사용자                |
| 이용 조건  | 비회원: 홈/카테고리 탐색만 가능<br>회원: 북마크, 마이페이지 이용 가능 |

### 서비스 목적 및 핵심 목표

* 사용자가 **운동형 / 지능형 / 예술형**으로 분류된 취미를 한눈에 탐색할 수 있도록 제공
* 관심 있는 취미를 **북마크**하여 개인 취향을 정리할 수 있도록 지원
* 복잡한 추천이나 커뮤니티 기능 없이, **명확한 탐색 → 선택 → 저장** 흐름에 집중

핵심 목표 요약:

* 취미 탐색의 진입 장벽 최소화
* 직관적인 카테고리 기반 정보 구조
* 최소 기능 기반의 명확한 사용자 경험 제공

## 3. 핵심 기능 및 사양 (Core Features and Specifications)

### 3-1. 전체 페이지 구조

| 구분      | 페이지       | 설명                   |
| ------- | --------- | -------------------- |
| Public  | 루트페이지     | 전체 취미 탐색, 카테고리 필터 제공 |
| Public  | 카테고리별 페이지 | 특정 카테고리 취미만 모아 제공    |
| Auth    | 로그인/회원가입  | 아이디·비밀번호 기반 인증       |
| Private | 마이페이지     | 북마크 목록 및 통계 확인       |

---

### 3-2. 루트페이지 (Home)

**구성 요소**

* Top Bar

  * 로고 (클릭 시 루트페이지 이동)
  * 카테고리 필터: 운동형 / 지능형 / 예술형
* Hero 섹션

  * 서비스 소개 문구 (취미 탐색 목적 명확히 전달)
* 메인 콘텐츠

  * 취미 카드 그리드
  * 카테고리 필터 선택 가능

**동작 사양**

| 항목    | 설명                     |
| ----- | ---------------------- |
| 기본 노출 | 전체 취미 카드 표시            |
| 필터 선택 | 선택된 대분류에 해당하는 취미만 표시   |
| 카드 클릭 | 상세 페이지 없음 (본 서비스 범위 외) |

---

### 3-3. 카테고리별 페이지

| 항목     | 설명                            |
| ------ | ----------------------------- |
| 상단 영역  | 선택된 카테고리명 + 간단한 소개 문구         |
| 콘텐츠 영역 | 해당 카테고리에 속한 취미 카드만 그리드 형태로 표시 |

카테고리 구성:

* 운동형
* 지능형
* 예술형

---

### 3-4. 취미 카드 공통 사양

| 요소      | 설명              |
| ------- | --------------- |
| 취미명     | 고정된 취미명 표시      |
| 카테고리 태그 | 운동형 / 지능형 / 예술형 |
| 북마크 버튼  | 로그인 상태에서만 활성화   |

---

### 3-5. 회원 관리 (로그인 / 회원가입)

| 항목      | 사양               |
| ------- | ---------------- |
| 인증 방식   | 아이디·비밀번호         |
| 회원가입 필드 | 아이디, 비밀번호        |
| 비회원 제한  | 북마크, 마이페이지 접근 불가 |

---

### 3-6. 마이페이지

**기능 구성**

* 북마크한 취미 리스트 확인
* 카테고리별 북마크 분포 통계 차트 제공

**통계 차트 사양**

| 항목    | 설명              |
| ----- | --------------- |
| 차트 유형 | 막대 그래프 또는 원형 차트 |
| 기준    | 운동형 / 지능형 / 예술형 |
| 데이터   | 사용자가 북마크한 취미 개수 |

## 4. 사용자 페르소나 및 시나리오 (User Persona and Scenarios)

### 페르소나 1: 취미 탐색형 사용자

* 나이: 20대 후반
* 목표: 새로운 취미 아이디어 탐색
* 특징: 회원가입 전 전체 카테고리 탐색 선호

**시나리오**

1. 루트페이지 접속
2. 카테고리 필터로 관심 영역 탐색
3. 흥미 있는 취미 목록 확인

---

### 페르소나 2: 취미 정리형 사용자

* 나이: 30대 초반
* 목표: 관심 취미를 정리하고 성향 파악
* 특징: 북마크 및 마이페이지 활용

**시나리오**

1. 로그인
2. 관심 취미 북마크
3. 마이페이지에서 북마크 목록 확인
4. 카테고리 분포 차트로 관심 성향 확인

## 6. 제한사항 (Limitations)

* 취미 상세 페이지 및 추천 기능 제공하지 않음
* 사용자 간 상호작용(댓글, 커뮤니티) 미제공
* 취미 목록 추가·수정 불가
* 외부 API 연동 없음
* 모바일 앱 미지원 (웹 전용)

---

## 7. 제안 추가 기능 (Suggested Additional Features)

* 본 프로젝트는 **강의 실습용**으로, 추가 기능을 제안하거나 구현하지 않음
* 본 섹션은 PRD 형식 충족을 위해 명시적으로 포함하되, 기능 확장은 범위 외로 정의함
