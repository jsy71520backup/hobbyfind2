## 3. 주요 단계 및 이벤트 흐름 (Main Steps and Flow of Events)

### UC-01. 메인페이지 탐색

1. 사용자가 홈(/)에 접속한다
2. 시스템은 전체 취미 카드 그리드를 표시한다
3. 사용자는 취미 목록을 스크롤하며 탐색한다

---

### UC-02. 카테고리 선택

1. 사용자가 카테고리 필터를 선택한다
2. 시스템은 선택된 카테고리의 취미만 표시한다
3. 사용자는 해당 취미 카드를 탐색한다

---

### UC-03. 북마크 추가/해제

1. 회원 사용자가 취미 카드의 북마크 버튼을 클릭한다
2. 시스템은 북마크 상태를 변경한다
3. 변경된 상태를 UI에 즉시 반영한다

---

### UC-04. 마이페이지 조회

1. 회원 사용자가 마이페이지로 이동한다
2. 시스템은 북마크 목록을 조회한다
3. 시스템은 북마크 통계 차트를 표시한다

---

### UC-05. 로그인

1. 사용자가 아이디/비밀번호를 입력한다
2. 시스템은 인증 정보를 검증한다
3. 성공 시 홈으로 이동한다

---

### UC-06. 회원가입

1. 사용자가 회원가입 정보를 입력한다
2. 시스템은 입력값 유효성을 검증한다
3. 회원가입 완료 후 로그인 페이지로 이동한다

---

## 4. 대체 흐름 및 엣지 케이스 (Alternative Flows and Edge Cases)

| 시나리오          | 처리 방식          |
| ------------- | -------------- |
| 비회원이 북마크 클릭   | 로그인 페이지로 이동    |
| 로그인 실패        | 에러 메시지 표시      |
| 마이페이지 비로그인 접근 | /login으로 리다이렉트 |
| 북마크 없음        | 빈 상태 메시지 표시    |

---

## 5. 사전조건과 사후조건 (Preconditions and Postconditions)

| Use Case | 사전조건   | 사후조건       |
| -------- | ------ | ---------- |
| 메인페이지 탐색 | 없음     | 취미 목록 표시   |
| 카테고리 탐색  | 없음     | 필터된 취미 표시  |
| 북마크 추가   | 로그인 상태 | 북마크 데이터 저장 |
| 마이페이지 조회 | 로그인 상태 | 북마크 정보 노출  |
| 로그인      | 계정 존재  | 인증 세션 생성   |
| 회원가입     | 없음     | 신규 계정 생성   |

---

## 6. 비즈니스 규칙 및 제약사항 (Business Rules and Constraints)

* 비회원은 북마크 및 마이페이지 기능 이용 불가
* 취미 목록은 사전에 정의된 고정 데이터만 사용
* 취미 상세 페이지는 제공하지 않음
* 카테고리는 3종으로 고정

---

## 7. 예외 처리 절차 (Exception Handling Procedures)

| 예외 상황     | 시스템 처리               |
| --------- | -------------------- |
| 인증 실패     | 오류 메시지 표시            |
| 세션 만료     | 자동 로그아웃 후 로그인 페이지 이동 |
| 데이터 조회 실패 | 기본 안내 메시지 표시         |

---

## 8. UI 고려사항 (User Interface Considerations)

* 카테고리 필터는 항상 상단에 노출
* 북마크 상태는 아이콘 변화로 즉시 인지 가능해야 함
* 로그인 에러는 폼 인라인 영역에 표시
* 통계 차트는 단순하고 직관적인 형태 사용

---

## 9. 데이터 요구사항 및 데이터 흐름 (Data Requirements and Data Flow)

### 주요 데이터 엔티티

| 엔티티      | 주요 필드                        |
| -------- | ---------------------------- |
| User     | userId, password             |
| Hobby    | hobbyId, hobbyName, category |
| Bookmark | userId, hobbyId              |

### 데이터 흐름 요약

* 취미 데이터 → 정적 데이터 로드
* 북마크 → 사용자별 저장 및 조회

---

## 10. 보안 및 개인정보 고려사항 (Security and Privacy Considerations)

* 비밀번호는 암호화 저장
* 인증 세션은 서버에서 관리
* 마이페이지 접근은 인증 사용자로 제한
* 불필요한 개인정보 수집 금지
